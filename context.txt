1. PROJECT OVERVIEW
================================================================================

OpenLabel is a consumer OSINT (Open-Source Intelligence) platform that unmasks
deceptive food labeling practices in India. It analyzes food product labels
using AI-powered forensic analysis to detect health-washing, sugar splitting,
economic fraud, and FSSAI compliance violations.

Architecture:
  Flutter App (Member 1) ↔ FastAPI Backend (Member 2) ↔ Gemini AI (Member 3) ↔ Supabase

The backend serves as the "Engine Room" — a FastAPI server that:
  • Accepts food label images (base64) or e-commerce URLs from the Flutter app
  • Processes them through Google Gemini 2.0 Flash AI
  • Enriches analysis with OSINT wholesale price data
  • Validates FSSAI license numbers
  • Stores all scan results in Supabase PostgreSQL
  • Returns structured JSON reports to the Flutter app

5. AUTHENTICATION SYSTEM
================================================================================

Authentication is handled via Supabase Auth with JWT tokens.

The Flutter app must:
  1. Sign up / Sign in users via Supabase Auth SDK (client-side)
  2. Obtain the JWT access token from the Supabase session
  3. Pass it as a Bearer token in the Authorization header:
     Authorization: Bearer <supabase_jwt_access_token>

Backend Auth Dependencies:
  • get_current_user   — REQUIRED auth. Returns user object or raises 401.
  • get_optional_user  — OPTIONAL auth. Returns user or None (for anonymous scans).

Scan endpoints use OPTIONAL auth (anonymous scans allowed).
Reports and Profile endpoints use REQUIRED auth.

6.2 SCAN — IMAGE ANALYSIS
────────────────────────────────────────────────────────────────────────────────

  POST /api/v1/scan/image
  
  Auth: Optional (Bearer token). Scans work without login.
        If authenticated, scan is saved to the user's history.
  
  Request Body (ScanImageRequest):
  {
    "image_base64": "<base64-encoded-image-string>",   // REQUIRED
    "product_name": "Patanjali Honey",                 // OPTIONAL
    "retail_price": 250.0                              // OPTIONAL (INR)
  }
  
  Response Body (ScanResponse):
  {
    "scan_id": "uuid-string",
    "product_name": "Patanjali Honey",
    "trust_score": "RED" | "YELLOW" | "GREEN",
    "overall_verdict": "Short summary of findings...",
    "flags": [
      {
        "title": "Sugar Splitting Detected",
        "description": "Multiple sugar variants found...",
        "category": "Sugar" | "Legal" | "OSINT" | "UPF" | "FSSAI"
      }
    ],
    "upf_score": 7,                                    // 1-10 scale, nullable
    "fssai_number": "10123456789012",                  // nullable
    "legal_draft_available": true,
    "legal_draft_text": "Formal complaint text...",    // nullable
    "created_at": "2026-03-30T12:00:00Z"               // nullable
  }
  
  Flow:
    1. Flutter sends base64 image
    2. Backend builds OSINT context (wholesale prices)
    3. Image + context sent to Gemini 2.0 Flash Vision API
    4. FSSAI number validated if extracted
    5. Results stored in Supabase 'scans' table
    6. Structured ScanResponse returned

6.3 SCAN — LINK/URL ANALYSIS
────────────────────────────────────────────────────────────────────────────────

  POST /api/v1/scan/link
  
  Auth: Optional (Bearer token)
  
  Request Body (ScanLinkRequest):
  {
    "url": "https://www.bigbasket.com/pd/some-product"  // REQUIRED
  }
  
  Response Body: Same ScanResponse as image scan
  
  Supported E-Commerce Sites:
    • BigBasket     (bigbasket.com)
    • Blinkit       (blinkit.com)
    • Amazon India  (amazon.in / amazon.com)
    • Flipkart      (flipkart.com)
    • Any URL       (generic fallback parser)
  
  Flow:
    1. Flutter sends product URL
    2. Backend scrapes the page (httpx + BeautifulSoup)
    3. Extracts: product name, brand, ingredients, nutrition, price, FSSAI, weight
    4. Builds OSINT context
    5. Scraped text sent to Gemini 2.0 Flash (text-mode)
    6. FSSAI validated, results stored, response returned
  
  Error: Returns 422 if scraping completely fails.

7. DATABASE SCHEMA (Supabase PostgreSQL)
================================================================================

────────────────────────────────────────────────────────────────────────────────
Table: scans
────────────────────────────────────────────────────────────────────────────────
  id                  UUID PRIMARY KEY (auto-generated)
  created_at          TIMESTAMPTZ (auto: now())
  user_id             UUID (FK → auth.users, nullable for anonymous scans)
  product_name        TEXT
  input_source        TEXT CHECK ('image' | 'link')
  raw_text_extracted  TEXT (scraped/OCR text, nullable)
  trust_score         TEXT CHECK ('RED' | 'YELLOW' | 'GREEN')
  full_report         JSONB (complete Gemini AI output)
  fssai_number        VARCHAR(14)

  Indexes:
    - idx_scans_user_id        (user lookup)
    - idx_scans_trust_score    (filter by trust score)
    - idx_scans_created_at     (date sorting DESC)
    - idx_scans_fssai          (FSSAI number lookup)
    - idx_scans_user_date      (composite: user + date DESC)

────────────────────────────────────────────────────────────────────────────────
Table: user_profiles
────────────────────────────────────────────────────────────────────────────────
  id                  UUID PRIMARY KEY (FK → auth.users)
  created_at          TIMESTAMPTZ (auto: now())
  updated_at          TIMESTAMPTZ (auto-updated via trigger)
  allergies           TEXT[] (string array, default: '{}')
  preference_level    TEXT CHECK ('Strict' | 'Casual'), default: 'Casual'

────────────────────────────────────────────────────────────────────────────────
Row Level Security (RLS):
────────────────────────────────────────────────────────────────────────────────
  scans:
    - Users can SELECT their own scans (user_id = auth.uid())
    - Users can INSERT their own scans (or anonymous with user_id = NULL)
    - Anonymous scans are not readable via client
  
  user_profiles:
    - Users can SELECT, INSERT, UPDATE their own profile only

────────────────────────────────────────────────────────────────────────────────
Triggers:
────────────────────────────────────────────────────────────────────────────────
  - on_auth_user_created → auto-creates user_profiles row on signup
  - set_updated_at       → auto-updates user_profiles.updated_at on update

────────────────────────────────────────────────────────────────────────────────
Dashboard Views (admin only):
────────────────────────────────────────────────────────────────────────────────
  - recent_scans_summary   — Last 100 scans with verdict + flag count
  - trust_score_stats      — Trust score distribution + unique users
  - most_flagged_products  — Products scanned >1 time, sorted by frequency

8. AI ENGINE — GEMINI 2.0 FLASH INTEGRATION
================================================================================

Model: gemini-2.0-flash
SDK:   google-genai (Python)

Two analysis modes:
  1. analyze_image() — Uses Gemini Vision API with base64 image + text prompt
  2. analyze_text()  — Uses Gemini text API with scraped product data

Configuration:
  - temperature: 0.2 (low for factual analysis)
  - response_mime_type: "application/json"
  - response_schema: OpenLabelReport (structured output enforcement)
  - system_instruction: TECH_JUSTICE_SYSTEM_PROMPT

The AI performs 6 analysis checks:
  1. Ingredient Splitting — Detects split-sugar deception (multiple sugar types)
  2. Economic Sniper      — Compares retail vs wholesale price for adulteration
  3. FSSAI Compliance     — Validates 14-digit license format
  4. UPF Risk             — Scores ultra-processed food additives (1-10)
  5. Legal Compliance     — Checks FSSAI 2026 labeling requirements
  6. Legal Draft          — Auto-generates "Jago Grahak Jago" complaint if RED

AI Output Schema (OpenLabelReport):
  {
    "trustScore": "RED|YELLOW|GREEN",
    "overallVerdict": "string",
    "productName": "string|null",
    "flags": [{"title": "...", "description": "...", "category": "..."}],
    "upfScore": int|null (1-10),
    "fssaiNumber": "string|null",
    "legalDraftAvailable": bool,
    "legalDraftText": "string|null"
  }

Flag Categories: Sugar, Legal, OSINT, UPF, FSSAI

Error Handling: On Gemini failure, returns a YELLOW fallback report with error
                message. The scan still succeeds — analysis is best-effort.

9. WEB SCRAPER — MULTI-SITE SUPPORT
================================================================================

The scraper extracts product data from Indian e-commerce sites.

Supported Sites (with dedicated parsers):
  • BigBasket  — Product name, brand, price, ingredients, FSSAI
  • Blinkit    — Product name, ingredients, nutrition, price, FSSAI
  • Amazon.in  — Title, brand, price, features, description, ingredients, FSSAI
  • Flipkart   — Title, price, FSSAI
  • Generic    — Fallback: h1/title, raw text, FSSAI, price

Scraper Capabilities:
  - Async HTTP with httpx (15s timeout)
  - Browser-like User-Agent headers
  - Follows redirects
  - FSSAI regex extraction: /\b(\d{14})\b/
  - Price regex extraction: /₹\s*([\d,]+(?:\.\d{2})?)/
  - Structured ScrapedProduct dataclass output

ScrapedProduct Fields:
  - product_name, brand, ingredients, nutrition_info
  - price (float, INR), weight, fssai_number
  - description, source_url, raw_text, errors

10. FSSAI VALIDATOR SERVICE
================================================================================

Validates 14-digit FSSAI license numbers with structural decoding.

Format: LLSSYYQQQNNNNNN
  - L  (digit 1):     License type (1=Central, 2=State, 3=Registration)
  - SS (digits 2-3):  State code (01-38, mapped to Indian states/UTs)
  - YY (digits 4-5):  Year of license grant
  - QQQ (digits 6-8): Quantity/category code
  - NNNNNN (digits 9-14): Unique registration number

Validation Checks:
  1. Must be exactly 14 digits
  2. License type must be 1, 2, or 3
  3. State code must match known Indian state/UT
  4. Year must be in reasonable range (2006-2030)

Output (FSSAIValidationResult):
  - is_valid, license_number, license_type, state, year_granted
  - errors (list of validation failure messages)
  - decoded_info (full breakdown dict)

Covers all 38 Indian state/UT codes.

11. OSINT WHOLESALE PRICE DATA
================================================================================

Used for the "Economic Sniper" feature — detects products sold below raw
material cost (indicating probable adulteration).

Data Source: data/wholesale_prices.json
  - 20 Indian food commodities with wholesale prices
  - Sources: APMC Market Data, NCDEX, Indian Commodity Exchange
  - Last updated: December 2025

Commodities Covered:
  Honey, Ghee (Cow/Buffalo), Olive Oil, Saffron, Coconut Oil, Mustard Oil,
  Turmeric, Red Chilli, Black Pepper, Cumin, Basmati Rice, A2 Milk,
  Almond, Cashew, Jaggery, Paneer, Tea, Coffee, Cinnamon

Rule: If retail_price_per_kg < wholesale_price_per_kg → "Probable Adulteration"

The OSINT context is formatted as a text string and passed to Gemini as part
of the analysis prompt, so the AI can use it for economic fraud detection.

13. FLUTTER INTEGRATION GUIDE
================================================================================

What the Flutter app needs to implement:

1. AUTHENTICATION (via Supabase Flutter SDK):
   - Email/password signup + login
   - Obtain JWT access token from Supabase session
   - Store and refresh tokens
   - Pass token in Authorization header for API calls

2. IMAGE SCAN FLOW:
   - Capture photo or pick from gallery
   - Convert to base64 string
   - POST to /api/v1/scan/image with base64 in body
   - Optionally include product_name and retail_price
   - Display the ScanResponse (trust score, flags, verdict, legal draft)

3. LINK SCAN FLOW:
   - Text input for URL or URL paste/share
   - POST to /api/v1/scan/link with URL in body
   - Display the same ScanResponse

4. SCAN HISTORY:
   - GET /api/v1/reports with auth token
   - Paginated list with trust_score color coding
   - Tap to GET /api/v1/reports/{scan_id} for full detail

5. USER PROFILE / ALLERGY SETTINGS:
   - GET /api/v1/profile to load current settings
   - PUT /api/v1/profile to save allergies and preference level
   - Allergy list: free-form string array
   - Preference: "Strict" or "Casual"

6. TRUST SCORE COLOR MAPPING:
   - RED    → Dangerous / Deceptive product (red color)
   - YELLOW → Caution / Some concerns (amber/yellow color)
   - GREEN  → Safe / Compliant product (green color)

7. FLAG CATEGORIES for UI icons/grouping:
   - Sugar  → Sugar-splitting deception
   - Legal  → Labeling compliance violations
   - OSINT  → Economic fraud / price anomaly
   - UPF    → Ultra-processed food markers
   - FSSAI  → License number issues

8. LEGAL DRAFT:
   - If legal_draft_available == true, show the legal_draft_text
   - This is a formal "Jago Grahak Jago" consumer complaint
   - Allow user to copy/share the text

14. DATA FLOW SUMMARY
================================================================================

Image Scan:
  Flutter → base64 image → POST /api/v1/scan/image
    → Backend builds OSINT context from wholesale_prices.json
    → Sends image + context to Gemini 2.0 Flash Vision
    → Gemini returns structured OpenLabelReport JSON
    → Backend validates FSSAI number (if extracted)
    → Stores scan in Supabase 'scans' table
    → Returns ScanResponse to Flutter

Link Scan:
  Flutter → URL → POST /api/v1/scan/link
    → Backend scrapes URL with site-specific parser
    → Extracts: name, ingredients, nutrition, price, FSSAI, description
    → Builds OSINT context
    → Sends scraped text + context to Gemini 2.0 Flash (text mode)
    → Validates FSSAI, stores, returns ScanResponse

History:
  Flutter → GET /api/v1/reports (with JWT)
    → Backend queries Supabase for user's scans
    → Returns paginated ReportListResponse

Detail:
  Flutter → GET /api/v1/reports/{id} (with JWT)
    → Backend fetches single scan (ownership verified)
    → Returns ReportDetailResponse with full raw data

Profile:
  Flutter → GET/PUT /api/v1/profile (with JWT)
    → Read/update user allergies and preference settings

================================================================================
15. ERROR HANDLING PATTERNS
================================================================================

  - AI failure:    Returns YELLOW fallback report (scan doesn't fail)
  - DB failure:    Scan still returns (analysis result not lost)
  - Scrape failure: Returns 422 with error message
  - Auth failure:   Returns 401 with "Invalid or expired authentication token"
  - Not found:     Returns 404 for missing scan reports
  - Server error:  Returns 500 with generic message (details logged)
